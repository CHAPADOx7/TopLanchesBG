Help do Sistema

Guia completo de operação: pedidos, status, fechamento de delivery, WhatsApp e relatórios.

Fluxo dos botões de pedido

Aceitar e Preparar: habilitado apenas quando o pedido está em Pendente. Ao confirmar entregador, o status vai para Em Preparo.

Automatizar comanda da cozinha: ao marcar essa opção no gerenciador, o sistema abre automaticamente a impressão da comanda de cozinha quando o pedido é aceito.

Comanda Cozinha: abre a versão reduzida para bobina térmica (80mm), pronta para impressão.

Saiu para Entrega: habilitado apenas quando o pedido está em Em Preparo.

Entregue: habilitado apenas quando o pedido está em Saiu para Entrega.

Após Entregue: os botões de transição ficam bloqueados, restando apenas impressão de documentos (NF/PDV e comanda).

Comanda Moto-boy: abre o documento da entrega para impressão imediata.

Nota / PDV: abre o comprovante da loja com resumo do pedido.

Fechamento de delivery

Ao clicar em Fechar Delivery, todos os pedidos saem do gerenciador ativo e são movidos para Pedidos Arquivados.

No mesmo fechamento, o contador visual de pedidos é zerado. Ao abrir novamente o delivery, a contagem começa do início.

Para consultar pedidos antigos: Pedidos Arquivados.

WhatsApp automático (Business) - guia 100% prático

Use este roteiro exatamente na ordem para habilitar mensagens automáticas de aceite, despacho e entrega.

1) Pré-requisitos

Tenha uma conta no Meta Business Manager e acesso de administrador ao negócio.

Tenha um número que possa ser usado no WhatsApp Business API (não pode estar ativo em WhatsApp comum no mesmo momento).

2) Criar/abrir app no Meta for Developers

Acesse developers.facebook.com/apps.

Clique em Create App, selecione tipo Business e finalize.

No menu do app, clique em Add Product e adicione WhatsApp.

3) Pegar o Phone Number ID (onde exatamente)

No painel do app da Meta: WhatsApp > API Setup.

Nessa tela, localize o campo Phone number ID e copie o valor numérico.

Esse valor deve ser colado no campo Phone Number ID do seu sistema.

4) Gerar token (teste e produção)

Token temporário (rápido para teste): na mesma tela WhatsApp > API Setup, copie o Temporary access token.

Token recomendado (produção): em business.facebook.com/settings, vá em Users > System Users, crie um usuário de sistema admin, atribua o app e gere token permanente com permissões do WhatsApp.

No sistema, cole esse token no campo Token de acesso.

5) Preencher no sistema (tela correta)

Abra Gerenciador de WhatsApp.

Preencha:

Ativar mensagens automáticas: marcado.

Token de acesso: token gerado na Meta.

Phone Number ID: valor da etapa 3.

Versão da API: mantenha v20.0 (ou a versão estável atual da Meta).

Clique em Salvar configurações.

6) Fazer teste real

Crie um pedido de teste e no admin clique em Aceitar e Preparar.

Depois avance para Saiu para Entrega e Entregue.

O cliente deve receber as mensagens em cada etapa.

7) Erros mais comuns e como resolver

Não chega mensagem: token expirado ou inválido. Gere novo token e salve novamente.

Erro de permissão: confirme se o token foi gerado com permissões do WhatsApp para o app correto.

Número errado: confirme se o Phone Number ID foi copiado da tela correta (WhatsApp > API Setup).

Formato de telefone do cliente: o sistema normaliza para Brasil, mas verifique se o cadastro do cliente está correto.

8) Checklist final (rápido)

[ ] Produto WhatsApp adicionado no app da Meta.

[ ] Phone Number ID copiado de WhatsApp > API Setup.

[ ] Token válido colado no sistema.

[ ] Opção de automação marcada no Gerenciador de WhatsApp.

[ ] Pedido de teste enviado com sucesso em todas as etapas.

Referência oficial: WhatsApp Cloud API.
